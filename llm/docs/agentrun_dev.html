一、AgentRun 生态概览

阿里云函数计算 AgentRun 是面向 AI Agent 应用的一站式部署与运维平台，提供从开发、部署到监控、评估的完整生命周期管理能力。

🚀 AgentRun 核心定位

AgentRun 是阿里云函数计算 FC 推出的 Agent 专属运行环境，专为大模型应用设计，提供：

  • 一键部署：支持代码包、容器镜像、OSS 多种部署方式
  • 弹性伸缩：根据流量自动扩缩容，应对高并发场景
  • 高并发支持：单实例最大 200 并发，轻松应对流量洪峰
  • 全链路可观测：LLM 调用、工具调用、性能指标实时监控
  • 内置评估：多维度评测框架，持续优化 Agent 质量

核心能力矩阵

📦部署管理

支持 Python/Node.js 等多种运行时，代码包上传、容器镜像、CI/CD 流水线集成

⚡弹性计算

毫秒级冷启动，定时伸缩策略，单实例最大 200 并发，按需付费

📊可观测性

LLM Token 消耗分析、工具调用统计、TTFT/TPOT 性能指标、错误率监控

🎯评估评测

内置安全性、工具使用能力等多维度评估器，支持自定义评估任务

为什么选择 AgentRun？
  • 高并发友好：专为 Agent 场景优化，支持单实例 200 并发，弹性扩缩容无缝衔接
  • 成本可控：按实际使用量计费，闲时自动缩容至最小实例数
  • 运维简单：全托管服务，无需管理基础设施，专注 Agent 业务逻辑

三、弹性扩缩容（核心能力）

弹性扩缩容是 AgentRun 的核心能力之一，也是应对高并发场景的关键技术方案。本章节重点介绍弹性策略配置和高并发场景下的实践方案。

🎯 弹性扩缩容核心价值

面对项目上线后的高并发场景，弹性扩缩容能够：

  • 自动应对流量洪峰：无需人工干预，系统自动扩容应对突发流量
  • 成本优化：低峰期自动缩容，避免资源浪费
  • 保障服务可用性：确保高峰期实例充足，避免服务降级
  • 精准定时调控：支持 CRON 表达式，按业务周期预配置伸缩策略

3.2 高并发场景配置建议

  1. 评估业务峰值分析历史流量数据，确定高峰时段和预估并发量，如周末午间 12:00-18:00
  2. 配置定时伸缩策略针对可预测的高峰时段，提前配置 CRON 策略，确保实例在高峰前就位
  3. 设置合理的最小实例数根据预估并发量和单实例并发度（200），计算所需最小实例数 = 预估并发 / 200
  4. 配置会话隔离对于需要保持状态的场景，启用 Cookie 或 HeaderField 会话隔离
  5. 监控与调优通过可观测性看板监控实际负载，持续优化弹性策略参数
最佳实践：
  • 时延敏感业务建议最小实例数 ≥ 1，保障毫秒级热启动
  • 促销活动前 30 分钟提前扩容，避免冷启动延迟
  • 配置多条策略覆盖不同场景，系统自动取最大值
  • 结合单实例最大并发（200）合理规划实例数量

测试目标：验证在不同并发压力下（50 → 100 → 300 → 500 → 1000），AgentRun 弹性扩缩容能力是否满足业务需求，重点关注 RPS、TTFT、TPOT 等核心指标的变化趋势。

压测解读：
  • 实例数弹性扩容 — 非瓶颈：AgentRun 弹性扩容能力充足，实例数随并发自动增长，未出现扩缩容延迟导致的性能下降
  • 工具调用耗时 — 非瓶颈：传统后端系统在 50→500 并发下增幅仅 10-70ms，表现稳定，工具链路未成为系统瓶颈
  • LLM — 瓶颈：现有模型 Quota 不足以支撑高并发请求，500 并发下 qwen3.6-plus 错误数 2.531K，需提升模型 QPS 配额以匹配弹性扩容能力
  • 50/100 并发下所有请求全部成功（5000/5000），LLM 和工具调用错误数均为 0
  • 300 并发下 4997/5000 成功（3 失败），plus 错误数 1.5K（部分请求重试）
  • RPS 随并发递增：6.21（50）→ 8.38（100）→ 10.83（300）→ 11.91（500）
  • 对话平均耗时在 5.55s（50）~ 9.03s（100）之间，300/500 并发下分别为 8.17s/8.60s
  • 1000 并发数据待补充，用于验证弹性扩容在极限压力下的表现
